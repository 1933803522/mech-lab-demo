# 力学实验低代码平台 AI 开发实验

这是一个用于复盘过往公司项目经验的个人实验仓库。项目不会一比一复刻原公司项目，而是抽象出“力学实验低代码平台”这一类产品问题，用 AI 参与从项目立项、需求梳理、PRD 编写、UI 设计到开发实现的完整流程。

这个仓库既是一个产品 Demo 的孵化过程，也是一次关于 AI 如何进入项目全生命周期的工作流实验。

## 项目目标

本项目主要沉淀两类产物：

1. **力学实验低代码平台 Demo**
   - 通过一个尽量完整的小闭环，验证核心产品假设。
   - 将过往项目经验转译为可展示、可迭代、可开发的产品原型。
   - 尝试用 AI 协助完成需求分析、产品设计、前端开发、测试与迭代。

2. **AI 项目协作工作流**
   - 记录 AI 在项目立项、PRD、原型/UI、开发、测试、复盘等阶段的使用方式。
   - 总结可复用的提示词、文档结构、评审清单和协作方法。
   - 观察 AI 在真实项目语境中的优势、边界和需要人工把关的环节。

## 项目定位

本仓库不是商业项目的源码迁移，也不是原业务系统的复刻。它更像是一个“复盘 + 重构 + 实验”的工作台：

- **复盘**：整理过去做项目时积累的业务理解、产品设计经验和工程经验。
- **重构**：将原项目中的通用问题抽象为一个新的、可公开沉淀的低代码平台 Demo。
- **实验**：系统性验证 AI 在项目生命周期中的参与方式，并沉淀可复用流程。

## 当前阶段

当前项目处于 **PRD 整理阶段**。由于完整项目范围较大，本阶段不会一次性写完完整 PRD，也不会直接进入复杂功能开发，而是先从简单页面开始，跑通一套可复用的 AI 协作流程。

当前重点工作包括：

- 先整理一部分简单页面的 PRD，明确页面目标、信息结构、交互逻辑和边界状态。
- 基于简单页面 PRD 生成 UI 方案或 UI 图。
- 再根据确认后的 UI 方案生成前端代码，完成“简单页面 PRD → UI → 前端代码”的小闭环。
- 在补充前端页面和交互的过程中，持续反向修正和完善 PRD。
- 在简单页面流程跑通后，再进入低代码平台、流程拖拽等复杂功能设计。
- 后端能力、mock 数据生成方式、后端语言和技术方案暂不作为当前重点，后续在 PRD 和前端形态更清晰后再慎重确定。

## 阶段路线图

### Phase 1：简单页面 PRD 整理

- 选择项目中相对独立、复杂度较低的页面。
- 使用 AI 辅助整理 PRD 内容，并同步沉淀 PRD 编写规范。
- 明确页面功能、字段、状态、交互和异常场景。

### Phase 2：简单页面 UI 生成

- 基于已整理的 PRD，让 AI 生成页面 UI 方案或 UI 图。
- 由人工判断 UI 是否符合业务目标、页面结构和交互预期。
- 将可复用的 UI 生成提示词和评审标准沉淀到工作流文档中。

### Phase 3：简单页面前端开发

- 基于 PRD 和 UI 方案，让 AI 生成前端代码。
- 人工检查代码结构、交互效果、可维护性和实现边界。
- 通过前端实现暴露 PRD 中不清晰、不完整或不合理的地方。

### Phase 4：PRD 迭代完善

- 在前端页面逐步补充的过程中，持续回填 PRD。
- 将页面字段、交互状态、数据结构、异常逻辑和业务规则写得更完整。
- 当 PRD 足够清晰后，再补充后端需要关注的接口、数据、权限和流程信息。

### Phase 5：复杂功能与后端方案设计

- 在简单页面流程和 PRD 结构稳定后，再设计低代码平台相关能力。
- 重点处理流程拖拽、组件配置、实验流程编排等复杂交互。
- 逐步补充后端接口设计、mock 数据生成方式和数据流转逻辑。
- 后端语言、框架和技术方案单独评估，不在早期阶段仓促决定。

## AI 与人的分工

本项目不是完全交给 AI 自动生成，而是以人机协作为核心：

- **人在前面做判断**：明确项目目标、业务边界、页面取舍和最终验收标准。
- **AI 辅助整理 PRD**：根据口述信息、项目背景和检查清单，帮助生成结构化 PRD，并总结 PRD 写作规范。
- **AI 辅助生成 UI**：根据 PRD 输出 UI 方案或 UI 图，再由人工判断是否符合产品预期。
- **AI 辅助生成代码**：根据确认后的 PRD 和 UI 方案生成前端代码，人工负责审查、调整和最终确认。
- **AI 辅助复盘流程**：在每个阶段记录有效提示词、问题、返工点和可复用工作流。

## 记录原则

- 对原公司项目经验做抽象和脱敏，不包含敏感业务信息、内部资料或不可公开内容。
- 先追求完整闭环，再逐步提高细节质量。
- 每个阶段都尽量留下可复用的文档、清单或工作流。
- 明确区分 AI 生成内容、人工判断和最终决策依据。
