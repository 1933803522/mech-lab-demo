# PRD写作基本要求清单


## 一、内容完整性要求（PRD里必须有什么）

### 1. 文档概述
- [ ] **项目背景**：用1-2段话说清楚"为什么要做这个项目"（解决了什么用户痛点/业务机会）
- [ ] **项目目标**：至少1个**可量化**的指标（如"支付成功率提升至75%"），并注明数据来源依据
- [ ] **项目范围**：
  - [ ] 本期包含哪些功能模块（明确列表）
  - [ ] 本期不包含哪些功能（明确列表，防止范围蔓延）

### 2. 产品结构
- [ ] **信息架构图**：用脑图/树状图展示页面层级关系
- [ ] **核心业务流程图**：用泳道图/流程图展示用户操作路径（至少覆盖主流程）

### 3. 功能需求详情（PRD最核心部分）
每个功能模块/页面必须包含：

- [ ] **页面/模块名称**：明确标识
- [ ] **页面布局描述**：从上到下的区域划分，每个区域展示什么内容
- [ ] **操作与交互**：用户能做什么操作？点击/滑动/输入后发生什么？
- [ ] **正常逻辑**：操作成功的完整路径
- [ ] **异常逻辑**（至少覆盖）：
  - [ ] 网络断开时显示什么
  - [ ] 接口报错时提示什么文案
  - [ ] 输入框格式错误时如何提示
  - [ ] 数据为空时页面显示什么
  - [ ] 快速重复点击时如何处理（防抖/防重复提交）
- [ ] **数据规则**：字数限制、格式要求、图片大小、列表分页规则等
- [ ] **权限要求**：是否需要登录？不同角色看到的是否不同？

### 4. 接口需求描述（给后端看的前后端契约）
每个涉及接口调用的功能必须包含：

- [ ] **接口用途**：这个接口用来做什么
- [ ] **请求方式**：GET/POST/PUT/DELETE
- [ ] **请求参数**：字段名、类型、是否必填、说明
- [ ] **成功返回**：返回什么数据，前端拿到后怎么处理
- [ ] **失败返回**：有哪些错误码，分别对应什么前端提示
- [ ] **特殊规则**：频次限制、缓存策略、超时处理等

### 5. 非功能性需求
- [ ] **性能需求**：页面加载时间、接口响应时间、并发支持数
- [ ] **兼容性需求**：支持的操作系统版本、屏幕尺寸范围
- [ ] **安全需求**：密码加密、敏感信息脱敏等
- [ ] **埋点需求**：哪些按钮/页面需要统计点击量或曝光量


## 二、写作规范要求（怎么写才合格）

### 1. 语言要求
- [ ] **拒绝形容词，只用数字和动词**
  - ❌ "尽量快" → ✅ "接口响应时间 ≤ 200ms"
  - ❌ "好看的弹窗" → ✅ "弹窗宽度为屏幕宽度的80%，圆角12px"
  - ❌ "友好提示" → ✅ "提示文案为'网络连接失败，请检查网络设置'"
- [ ] **主语明确**：谁（哪个角色）在什么条件下执行什么操作
- [ ] **无歧义**：同一概念全文使用同一术语（如统一叫"商品"不叫"产品"）

### 2. 结构化要求
- [ ] **善用表格**：复杂逻辑、多状态对比优先用表格，不用大段文字
- [ ] **层级清晰**：使用标准Markdown标题层级（# ## ### ####）
- [ ] **条目化**：每个功能点独立成条目，便于开发和测试逐条对照

### 3. 可视化要求
- [ ] **流程图优于纯文字**：人脑对图形理解速度远快于文字
- [ ] **线框图辅助说明**：不要求高保真，但关键交互元素的位置要标出


## 三、版本管理要求（怎么迭代才不乱）

### 1. 版本号规则
- [ ] **V1.0.1**：错别字、文案修改、极小调整
- [ ] **V1.1、V1.2**：新增1-2个小功能（工作量 < 3天），在**同一文档内追加**
- [ ] **V2.0、V3.0**：新增独立大模块或架构级变更，**新建独立文档**

### 2. 修订历史（必填！）
- [ ] 每次修改必须记录：**修改日期、版本号、修改内容摘要、修改人**
- [ ] 修订历史表格放在文档最末尾或最开头（团队统一即可）


## 四、评审前自检要求（写完自己先过三遍）

### 第一遍：以小白用户身份走正常流程
- [ ] 所有主流程操作路径是否完整？（从入口到结束，每一步都写清楚了吗？）

### 第二遍：以"搞破坏"的身份走异常流程
- [ ] 断网、弱网时页面表现写了吗？
- [ ] 输入错误格式、空值、超长内容时提示写了吗？
- [ ] 快速双击、重复提交时防重复写了吗？
- [ ] 数据为空、列表到底、商品下架等边界状态写了吗？

### 第三遍：盯着"边界值"和"数字"
- [ ] 所有数字都有依据吗？（不是拍脑袋写的吗？）
- [ ] 所有"尽量""大概""差不多"都改成具体数值了吗？
- [ ] 字数限制、图片大小、分页条数等边界值都写了吗？
